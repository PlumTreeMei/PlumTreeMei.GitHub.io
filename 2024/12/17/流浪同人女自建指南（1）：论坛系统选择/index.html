流浪同人女自建指南（1）：论坛系统选择

发表于 2024-12-17 更新于 2024-12-21

前言

稍微整理了一下市面上常见的几种论坛系统软件。本人非专业程序员，因此会站在普通用户（即不需要什么代码知识预算也不高的前提下）分析他们的利弊。

1.Discuz！

Discuz！是目前国内使用最多、接受度最高、支持也最多的系统。大家所熟悉的比如随缘居之类的论坛都使用了 discuz。如今，想要开发一个基于 discuz! 的论坛非常方便，你可以在官网上下载它的源代码，也可以花点钱找专业人士帮你量身定制。然而，discuz! 并非传统意义上的开源软件，社区生态并不是很好，小白容易被坑。它目前的版权全部属于腾讯，因此在使用的时候也需要遵循由腾讯拟定的协议。目前市面上最新的免费版本 为 dsicuz！ X3.5。

指标 评价与说明
技术框架 PHP + MySQL
插件种类 非常多，但大部分需收费且质量良莠不齐
功能性 内置门户与积分系统等
安全性 官方停止更新，依赖社区修复漏洞与发布非官方更新版本，容易被攻击
迁移能力 插件兼容性差，迁移难度较高
维护难度 技术栈比较过时，缺少文档和官方维护。但国内相关从业者多，有钱可以兜底。
移动端支持 需插件与模板适配，不支持 PWA，需自行开发 App
用户体验 比较陈旧、但国内用户相对比较习惯
推荐最低硬件要求 1 核 CPU 512MB RAM 10GB 存储

2.Discourse

Discourse 是一个对服务器要求较高的开源论坛系统，适合规模大的社区。它的功能强大、免费插件多、界面也非常现代化、在外网国外有非常活跃的社区环境和良好的生态。但是国内目前使用 discourse 的人并不多，因此想要使用 discourse 论坛只能自行学习搭建或者使用官方的托管服务。然而，官方的托管服务月费要起码 100 刀以上还有诸多限制，对于同人论坛来说不是什么好的选择。至于自建论坛的话，则需要比较多的计算机相关知识和较高的英语水平，二次开发难度较大。

指标 评价与说明
技术框架 Ruby on Rails + Ember.js + PostgreSQL
插件种类 有非常多免费插件和模板
功能性 积分系统比较简陋，但有徽章系统、信任等级机制和内置聊天群组
安全性 有自动备份、SSL 证书自动更新、官方持续维护、有比较稳定的自动更新机制
迁移能力 支持多个平台的迁移，工具完善
维护难度 国内讨论少且涉及的技术栈在国内比较冷门。文档丰富但杂，需较高学习能力
移动端支持 响应式设计，支持 PWA
用户体验 设计现代化，国内用户需适应使用方式
推荐最低硬件要求 2 核 CPU 4GB RAM 20GB 存储

3.NodeBB

NodeBB 是基于 Node.js 构建的一个开源论坛软件。它同样拥有丰富的免费插件和现代化的界面，性能也不错。不过相对于 discourse 来说它的社区规模并没有那么大，官方提供的托管服务也特别昂贵。官方维护还算比较积极，但是对中文内容的支持似乎并不是很好。如果本身对 NodeBB 的技术栈比较熟悉，可以尝试看看。

指标 评价与说明
技术框架 Node.js + Express.js+ MongoDB
插件种类 有很多免费插件和模板
功能性 有内置的实时聊天功能
安全性 官方持续维护
迁移能力 有社区插件支持
维护难度 二次开发门槛比 Discourse 低、技术栈相对热门些
移动端支持 响应式设计，支持 PWA
用户体验 设计现代化，国内用户需适应使用方式
推荐最低硬件要求 2 核 CPU 2GB RAM 20GB 存储

4.Flarum

Flarum 是一款轻量级的开源论坛软件，对服务器的要求比较低。它的界面非常现代化和简洁，具有高效的性能和灵活的插件支持。功能虽然不多但是胜在非常精简，使用起来容易上手。特别适合要求不高的小型社区。

指标 评价与说明
技术框架 PHP + Laravel + MySQL
插件种类 插件灵活但生态仍在成长中
功能性 功能比较精简
安全性 官方持续维护
迁移能力 比较有限
维护难度 维护简单，适合 PHP 开发者
移动端支持 响应式设计，但没有 PWA
用户体验 设计精简，上手容易
推荐最低硬件要求 1 核 CPU 1GB RAM 10GB 存储
